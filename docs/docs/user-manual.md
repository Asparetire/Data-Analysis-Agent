# 用户手册

## 注册与登录

1. 打开应用首页，点击「注册」
2. 输入邮箱和密码（密码长度无硬性要求，但建议 ≥12 位）
3. 注册成功后自动登录，跳转到工作台
4. 下次访问直接用邮箱密码登录

!!! note "Token 机制"
    - Access token 15 分钟过期，前端自动用 refresh token 续期
    - Refresh token 7 天过期，每次 refresh 轮换 + 旧 token 立即吊销
    - 登出会吊销当前 refresh token，被偷的 token 立即失效

## 上传数据

1. 工作台左侧点「上传」
2. 拖拽或选择 CSV / Excel / JSON 文件（≤50MB）
3. 上传完成后，数据源出现在左侧列表，文件名作为默认显示名

支持的格式：

| 格式 | 后缀 | 说明 |
|------|------|------|
| CSV | `.csv` | 自动推断分隔符 |
| Excel | `.xlsx` / `.xls` | 多 sheet 会建多张表 |
| JSON | `.json` | 数组对象，字段从首元素推断 |

!!! tip "PII 脱敏"
    上传时 email / 手机号 / 身份证 列会被自动掩码后写入 SQLite。
    原始数据不落盘，只存掩码后的版本。

## 对话查询

1. 在数据源列表选择一个数据源（首次对话会绑定会话与数据源）
2. 在输入框提问，例如：
    - "总结这份数据"
    - "按销售额排序前 10 名"
    - "每个月的销售趋势，画折线图"
3. 回车发送，Agent 流式返回：
    - **token**：LLM 输出的文本增量
    - **tool_call**：Agent 调用工具（query_database / get_table_schema / create_chart）
    - **tool_result**：工具返回结果
    - **chart**：ECharts option，前端直接渲染图表
    - **end**：本轮对话结束

!!! warning "会话绑定"
    一个会话只能绑定一个数据源。切换数据源会触发 403，前端会自动
    重置会话。这是为了防止 Agent 把不同数据源的表混在一起查。

## 数据源管理

- **重命名**：数据源列表右键 → 重命名（修改 display_name，不影响文件名）
- **预览**：点击数据源，右侧显示前 5 行
- **schema**：查看字段名和推断类型
- **分页浏览**：底部翻页，支持自定义 `offset/limit/sort/dir`
- **多表浏览**：Excel 多 sheet 的数据源，下拉切换表
- **删除**：数据源列表右键 → 删除（同时删除 SQLite 文件 + 关联会话 + 元数据）

!!! danger "删除不可恢复"
    删除数据源会同时删除该数据源的所有会话历史。如有重要对话，
    先复制保存。

## 查询历史 (lineage)

每个数据源都记录查询历史：

- 点击数据源 → 「查询历史」标签
- 每条记录包含：时间、SQL、行数、是否成功
- 用于审计和回溯

## 已知限制

- SQL 只能 `SELECT` / `WITH`，禁止 `INSERT/UPDATE/DELETE/DROP`
- 结果限制 100 行
- 上传文件 ≤50MB
- 单表 SQL，不支持跨数据源 JOIN
- 没有 Python REPL，分析全部走 SQL
